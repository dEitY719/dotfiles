# JIRA Story 최종 초안 - AI Enable 파트 평가 Structure (2026)

> **평가 기간**: 2026년 4월 ~ 10월
> **평가 기준**: 파트(PL 평가 로직) + 파트원(팀원 평가 로직)
> **구조**: Epic(팀 미션, 생성 안 함) → Story(평가 항목) → Task(세부 활동, 추후 생성)
> **목적**: 팀→그룹→파트→파트원 목표 연관관계를 Structure로 가시화

---

## 0. 팀 미션과 연결 구조

> Epic은 팀 미션이므로 직접 생성하지 않으나, 모든 Story가 아래 팀 목표에 정렬(align)되어야 합니다.

### 팀 핵심 미션 (Epic 레벨 — 참조용)

| # | 팀 미션 | 설명 |
|---|---------|------|
| M1 | AI 활용 극대화 | 사업부 SW 경쟁력 강화 및 7000명 아바타 창출 |
| M2 | SW 품질 Shift-Left | 과제 상용화 인력 효율화에 기여 |
| M3 | AI 서비스/솔루션 제공 | 사업부에 필요한 AI service 제공, 홍보, 가이드 |
| M4 | SW Governance | 사업부 내 SW 과제의 Governance 담당 |

### 계층 연결 구조 (Structure 설계)

```
[팀 Epic] M1: AI 활용 극대화
  └─ [그룹 Story] (그룹장이 생성)
      └─ [파트 Story] AI Enable 파트 핵심과제 달성
          ├─ [파트원 Story] 홍길동 - 핵심과제 수행
          ├─ [파트원 Story] 김철수 - 핵심과제 수행
          └─ ... (파트원 10명)

[팀 Epic] M2: SW 품질 Shift-Left
  └─ [그룹 Story] (그룹장이 생성)
      └─ [파트 Story] AI Enable 파트 핵심과제 달성
          └─ ... (동일 파트 Story에 복수 미션 연결 가능)
```

### 평가 항목별 연결 매핑

| 평가 항목 | 비중 | 팀 미션 연결 | 파트 Story | 파트원 Story |
|-----------|------|-------------|-----------|-------------|
| 조직목표달성 | 50% | M1~M4 직접 연결 | 파트 OKR → 팀 목표 정렬 | 개인 OKR → 파트 OKR 정렬 |
| 혁신/개선업무 | 20% | M1, M3 간접 연결 | 파트 프로세스 개선 | 개인 Tool/AI 기여 |
| 조직시너지 창출 | 20% | M3, M4 간접 연결 | 타 조직 협업 | 동료 기여, 데이터 공유 |
| 역량강화 | 10% | 전체 미션 지원 | 부서원 역량 관리(PL) | 개인 역량 개발(팀원) |

## 1. 파트 레벨 Story (AI Enable 파트 — PL 평가 로직 적용)

> PL 평가 로직: 핵심과제 **3개 이하**, OKR **임원 승인**, 부서원역량강화는 **SCI 참조**

### Story P-1: 조직목표달성 (50%)

**Summary**: `[AIE 파트] 2026 조직목표달성 - OKR 기반 핵심과제 달성`

**Linked Mission**: M1, M2, M3, M4

**Description**:

```
h2. 목적
AI Enable 파트의 2026년 핵심과제를 OKR 프레임워크로 정의하고,
사업부 SW 경쟁력 강화 및 AI 활용 극대화에 기여한다.

h2. 평가 기준 (비중 50%)
- 핵심과제 3개 이하로 설정 (PL 기준)
- OKR 구성: Objective → Key Result → Initiative → KPI
- KPI는 측정 가능한 숫자로 정의
- OKR은 임원 승인 필수
- 최소 월 1회 1:1을 통한 진행상황 점검 및 피드백

h2. 팀 목표 정렬
|| 팀 미션 || 파트 Objective (예시) || KR (예시) ||
| M1: AI 활용 극대화 | AI 솔루션을 통한 7000명 아바타 기여 | AI 솔루션 적용 과제 N건 달성 |
| M2: SW 품질 Shift-Left | Shift-Left 활동으로 상용화 효율화 | 과제별 결함 조기 발견율 X% 향상 |
| M3: AI 서비스 제공 | 사업부 AI 서비스 제공 및 가이드 | AI 가이드 배포 N건, 활용률 X% |

h2. 운영 방식
- 4월: OKR 수립 및 임원 승인
- 매월: 1:1 feedback (진행률, 장애요인, 조정사항)
- 분기별: KPI 달성률 리뷰

h2. 성과 측정
- KPI 달성률 (정량)
- 과제 영향도 — 사업부 기여도 (정성)
- 일정 준수 및 실행력

h2. 기간
2026.04 ~ 2026.10
```

## 4. JIRA Structure 구성 가이드

### 최종 Structure 트리

```
SW 혁신팀 (팀 레벨 — Epic 참조만)
│
├─ [그룹] OO그룹 (그룹장이 생성)
│   │
│   ├─ [AIE 파트] 조직목표달성 (P-1)          ← 파트 Story
│   │   ├─ [홍길동] 조직목표달성 (M-1)         ← 파트원 Story
│   │   │   ├─ Task: 4월 OKR 수립              ← 추후 생성
│   │   │   ├─ Task: 5월 KPI 중간점검
│   │   │   ├─ Task: 6~7월 핵심과제 중간리뷰
│   │   │   ├─ Task: 8~9월 KPI 달성률 점검
│   │   │   └─ Task: 10월 최종 성과 정리
│   │   ├─ [김철수] 조직목표달성 (M-1)
│   │   ├─ [이영희] 조직목표달성 (M-1)
│   │   └─ ... (파트원 10명)
│   │
│   ├─ [AIE 파트] 혁신/개선업무 (P-2)
│   │   ├─ [홍길동] 혁신/개선업무 (M-2)
│   │   ├─ [김철수] 혁신/개선업무 (M-2)
│   │   └─ ...
│   │
│   ├─ [AIE 파트] 조직시너지 창출 (P-3)
│   │   ├─ [홍길동] 조직시너지 창출 (M-3)
│   │   └─ ...
│   │
│   └─ [AIE 파트] 부서원역량강화 (P-4)
│       ├─ [홍길동] 개인역량강화 (M-4)
│       └─ ...
```

### Structure 활용 팁

1. **필터링**: Summary에 `[AIE 파트]` 또는 `[{성명}]` 접두어가 있어 JQL 필터 용이
   - 파트 전체: `summary ~ "[AIE 파트]"`
   - 특정 파트원: `summary ~ "[홍길동]"`
   - 특정 평가항목: `summary ~ "조직목표달성"`

2. **계층 추적**: Structure에서 파트 Story를 펼치면 해당 항목에 기여하는 모든 파트원이 한눈에 보임

3. **균형 확인**: 각 파트 Story 아래 파트원 수가 고르게 분포되어 있는지 확인 → 팀 미션에 골고루 기여하고 있는지 검증 가능

4. **PL vs 팀원 구분**: 파트 Story는 PL 평가 로직(핵심과제 3개, 임원승인), 파트원 Story는 팀원 평가 로직(핵심과제 2개, 파트장승인) 적용

## 5. 다음 단계 (Task 생성 시)

각 파트원 Story 하위에 4월~10월 기간 동안 **5개 Task**를 생성:

```
예시) [홍길동] 조직목표달성 Story 하위:
  - Task 1: [4월] OKR 수립 및 파트장 승인
  - Task 2: [5월] 1차 KPI 중간 점검 및 1:1 feedback
  - Task 3: [6~7월] 핵심과제 중간 리뷰 및 방향 조정
  - Task 4: [8~9월] KPI 달성률 점검 및 추가 Initiative 실행
  - Task 5: [10월] 최종 성과 정리 및 평가 자료 제출

예시) [홍길동] 혁신/개선업무 Story 하위:
  - Task 1: [4월] 개선 대상 업무 분석 및 선정
  - Task 2: [5~6월] Tool/자동화 개발 1차
  - Task 3: [7월] 중간 점검 및 효과 측정
  - Task 4: [8~9월] Tool 고도화 및 확산
  - Task 5: [10월] 최종 개선 효과 보고서 작성
```
