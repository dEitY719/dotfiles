# 동료 답변(제안)
• 사용 중인 헬퍼 함수 스타일을 참고해, 함수명은 일관성을 위해 밑줄(_) 대신 대시(-) 사용을 권장합니다. 일반적으로 자바스크립트나 파이썬에서는 밑줄이 더 흔하지만, 프로젝트 내 스타일 가이드에 맞춰 통일하는 게 가장 중요합니다. 일관된 네이밍 표준을 정해 팀 내 공유하는 것을 추천합니다.

- *-help처럼 일관된 패턴을 쓰면 기억하기 쉽고 자동완성/검색(rg, compgen)에도 좋습니다. 현재 목록에 gc_help, litellm_help, mytool_help처럼 _help가 섞여 있어서 정리 대상이 맞습니다.
- 최근 CLI/alias 관행은 대체로 대시(-)를 더 자주 씁니다. 이유: 1) POSIX/프로그램 플래그와 동일한 감각, 2) zsh/bash 자동완성에서 자연스러운 분리, 3) 단어 경계 가독성. 다만 Bash 함수 이름은 대시를 허용하지 않으므로, 함수명은 snake_case, 호출용 alias만 대시 형태로 두는 식이 흔합니다.
- 나도 name-help를 추천합니다. 함수는 name_help로 두고, 사용자 노출 alias는 name-help로 매핑하는 형태가 안전하고 일관됩니다. 원하는 경우 전체를 다음처럼 표준화하면 좋습니다: 함수 foo_help + alias foo-help.
