### 🎯 Architecture Decisions

#### Why Dual Cleanup System?
1. **bash/setup.sh cleanup** - Fixes immediate state
   - Runs once during dotfiles initialization
   - Cleans ~/.zshrc of broken references

2. **~/.zshrc auto-cleanup** - Provides ongoing protection
   - Runs at every zsh startup
   - Defense-in-depth against future issues
   - Fast and silent operation

#### Why Installation Before Registration?
- Previous attempts registered plugin before installing
- Led to "plugin not found" errors at zsh startup
- New flow: Clone → Register → Configure (correct order)

#### Plugin Not Tracked in Git
- Users run install-* commands in their environment
- Each PC gets plugins independently
- No value in version control (local installations)
- Added to .gitignore to prevent accidental commits

---

### 🔧 Technical Details

#### Install Script Flow
```
1. Pre-flight checks
   ├─ Oh-My-Zsh installed?
   ├─ Plugin already installed?
   └─ Already registered in ~/.zshrc?

2. Cleanup (if needed)
   └─ Remove broken plugin references

3. Installation
   ├─ Clone from GitHub
   └─ Register in plugins array

4. Configuration
   └─ Add key bindings and settings
```

#### Auto-Cleanup in ~/.zshrc
```bash
# Runs at zsh startup
_dotfiles_auto_cleanup_plugins() {
    for plugin in zsh-autosuggestions zsh-syntax-highlighting ...
```
