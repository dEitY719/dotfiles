Scripts Overview

두 개의 스크립트, 각각 다른 역할

Script Purpose Type When
./setup.sh Shell 환경 설정 Required Initial install
./install.sh Claude / PG 보조 설정 Optional Optional

New User Setup

3단계로 완료하는 초기 설치

Step 1

Environment Setup

setup.sh 실행으로 환경 선택 (Public / Internal / External)

Step 2

Claude / PG Setup

install.sh 실행으로 Claude 설정 및 PostgreSQL 구성

Step 3

Restart Shell

exec bash 또는 exec zsh로 새 shell 진입

절대 삭제하면 안 되는 파일

  • shell-common/setup.sh — 환경별 설정 필수
  • bash/setup.sh — bash 환경변수 설정 필수
  • zsh/setup.sh — zsh 초기화 필수
  • git/setup.sh — git config symlink 필수
  • setup.sh (루트) — 위 파일들의 orchestrator

Each File's Role

각 파일이 수행하는 정확한 작업

shell-common/setup.sh

Environment Manager

사용자에게 3가지 환경 선택 요청, proxy.local.sh / security.local.sh 생성

Special: 환경별 설정 (Public/Internal/External)
bash/setup.sh

Bash Initializer

DOTFILES_BASH_DIR, SHELL_COMMON 환경변수 설정, ~/.bashrc symlink 생성

Special: 환경변수 설정 (필수!)
zsh/setup.sh

Zsh Initializer

~/.zshrc symlink, ~/.p10k.zsh symlink, 완료 메시지 출력

Special: 사용자 안내, p10k 설정
git/setup.sh

Git Config Manager

~/.gitconfig symlink 생성, UX 함수를 이용한 피드백

Special: UX 피드백
install.sh

Claude / PG Setup

Claude 설정, PostgreSQL 설정, Bash/Zsh/Git symlink 재설정

Note: Claude, PG 추가 설정만 가능
setup.sh (root)

Orchestrator

순서대로 모든 shell 및 환경 setup 실행

Side effect: 없음 (하위 스크립트 호출만)

Recommended Execution Order

상황별 올바른 실행 방법

New Users

처음 설치하는 사용자를 위한 순서

$ cd ~/dotfiles
$ ./setup.sh
# Step 1: Shell 설정
$ ./install.sh
# Step 2: Claude/PG 설정
$ exec bash

Existing Users

업데이트만 필요할 때

$ cd ~/dotfiles
$ ./install.sh
# Claude/PG 설정만 업데이트

Specific Shell

특정 shell만 재설정

$ ./bash/setup.sh
# Bash만 재설정
$ ./zsh/setup.sh
# Zsh만 재설정
$ ./git/setup.sh
# Git만 재설정

Troubleshooting

일반적인 문제와 해결 방법

ux_header: command not found 오류

원인: ux_lib.sh가 로드되지 않음

해결:

  1. zsh/zshrc 98번 줄 확인: $SHELL_COMMON/tools/ux_lib/ux_lib.sh 경로 확인
  2. bash/main.bash에서 ux_lib.sh 로드 확인
~/.bashrc 또는 ~/.zshrc symlink 깨짐

원인: setup.sh를 실행하지 않았거나, symlink가 잘못 설정됨

해결:

./setup.sh 다시 실행

새 shell에서 alias/function 로드되지 않음

원인: 기존 ~/.bashrc 또는 ~/.zshrc 파일이 존재하거나, setup.sh를 실행하지 않음

해결:

./setup.sh — setup.sh가 symlink를 생성하면서 기존 파일을 백업함

Core Principles

이해해야 할 5가지 핵심 원칙

01

Complementary Relationship

setup.sh와 install.sh는 보완 관계 — setup.sh: Shell 환경, install.sh: Claude/PG

02

Special Initialization

setup.sh의 하위 파일들은 특수 초기화 수행 — bash: 환경변수, zsh: 피드백

03

Symlink Only

install.sh는 symlink 재설정만 가능 — bash/zsh 환경변수 설정 안 함

04

Never Delete

setup.sh 파일들 절대 삭제하면 안 됨 — 특수 초기화 로직이 손실됨

05

Recovery Impossible

install.sh로 복구 불가능 — 초기 설치 시 setup.sh 필수

Quick Reference

핵심 디렉토리 구조

dotfiles/
├── bash/          # bash 셸 초기화
├── zsh/           # zsh 셸 초기화  
├── git/           # git 설정
├── shell-common/  # bash/zsh 공유 리소스
│   ├── aliases/   # 명령어 aliases
│   ├── functions/ # shell 함수
│   ├── env/       # 환경 설정
│   └── tools/     # 유틸리티 도구
├── setup.sh       # Orchestrator (필수)
└── install.sh     # Claude/PG (선택)
