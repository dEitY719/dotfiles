# using-git-worktrees
## 개요
안전한 분리 작업공간(worktree) 생성 및 초기 검증을 표준화하는 스킬

## 언제 사용하나
- 기존 작업공간과 분리된 구현 세션이 필요할 때
- 병렬 작업/리스크 높은 변경을 격리하고 싶을 때

## 핵심 절차(번역)
1. 디렉터리 후보와 ignore 상태를 먼저 검증한다.
2. 프로젝트명 기반으로 새 브랜치+worktree를 생성한다.
3. 프로젝트 setup과 베이스라인 테스트를 통과시킨 뒤 작업을 시작한다.

## 주의/금지 사항(번역)
- ignore 검증 없이 worktree 생성
- 기본 브랜치 확인 없이 분기
- 초기 테스트 실패 상태로 작업 진행

## 실행 체크리스트
- 작업 시작 전에 이 스킬이 현재 과제에 실제로 필요한지 확인한다.
- 필요하면 해당 스킬의 원문 `SKILL.md`에서 세부 규칙을 다시 확인한다.
- 진행 중에는 체크리스트/게이트 조건을 생략하지 않는다.
- 완료 보고 전에는 관련 검증 명령과 결과를 함께 남긴다.

## 빠른 예시
> "이번 기능은 분리해서 작업하고 싶어. using-git-worktrees 절차로 새 worktree 만들어줘."
