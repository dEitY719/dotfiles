# brainstorming
## 개요
구현 전에 사용자 의도와 요구사항을 설계로 구체화하는 사전 브레인스토밍 스킬

## 언제 사용하나
- 새 기능/컴포넌트/동작 변경 등 창의적 설계가 필요한 모든 작업 시작 시
- 요구사항이 크거나 모호해서 범위 분해가 필요한 경우

## 핵심 절차(번역)
1. 코드 작성 전에 반드시 설계 합의부터 진행한다.
2. 질문은 한 번에 하나씩 던져 목적/제약/성공 기준을 명확히 한다.
3. 2~3개 접근안을 비교하고 권장안을 제시한다.
4. 설계를 문서화하고 사용자 리뷰 게이트를 통과한 뒤에만 `writing-plans`로 넘어간다.

## 주의/금지 사항(번역)
- 설계 승인 없이 구현 스킬 호출
- 한 메시지에 질문 여러 개 묶기
- 설계 문서 없이 바로 구현 착수

## 실행 체크리스트
- 작업 시작 전에 이 스킬이 현재 과제에 실제로 필요한지 확인한다.
- 필요하면 해당 스킬의 원문 `SKILL.md`에서 세부 규칙을 다시 확인한다.
- 진행 중에는 체크리스트/게이트 조건을 생략하지 않는다.
- 완료 보고 전에는 관련 검증 명령과 결과를 함께 남긴다.

## 빠른 예시
> "새 결제 온보딩 플로우를 추가하고 싶어. brainstorming 절차로 요구사항 질문부터 시작해줘."

## 비고
- 이 문서는 팀 학습용 한국어 번역 가이드다.
- 원문 전체의 문장 단위 직역보다, 실무 적용 시 필요한 규칙/절차를 한국어로 명확하게 정리했다.
