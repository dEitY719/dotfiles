# receiving-code-review 스킬 한국어 번역 가이드

## 개요
코드 리뷰 피드백을 방어적으로 처리하지 않고 정확히 반영/응답하는 스킬

## 언제 사용하나
- 리뷰 코멘트를 받고 대응해야 할 때
- 외부 리뷰어와 합의 기반 커뮤니케이션이 필요한 때

## 핵심 절차(번역)
1. 피드백을 먼저 정확히 이해하고 사실 관계를 검증한다.
2. 유효한 지적은 수용하고 수정 계획/결과를 명확히 답한다.
3. 불명확하거나 오해 소지가 있으면 근거와 함께 정중히 재질문한다.

## 주의/금지 사항(번역)
- 감정적 반응/즉각 반박
- 근거 없는 "이미 해결됨" 응답
- YAGNI 위반 기능을 방어 논리로 유지

## 실행 체크리스트
- 작업 시작 전에 이 스킬이 현재 과제에 실제로 필요한지 확인한다.
- 필요하면 해당 스킬의 원문 `SKILL.md`에서 세부 규칙을 다시 확인한다.
- 진행 중에는 체크리스트/게이트 조건을 생략하지 않는다.
- 완료 보고 전에는 관련 검증 명령과 결과를 함께 남긴다.

## 빠른 예시
> "리뷰 코멘트가 여러 개 달렸어. receiving-code-review 원칙으로 코멘트별 대응안을 작성해줘."
