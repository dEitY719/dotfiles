# receiving-code-review

## 한 줄 설명
코드 리뷰 피드백을 방어적으로 처리하지 않고 정확히 반영/응답하는 스킬

## 이 스킬이 해결하는 문제
- 작업 흐름에서 자주 발생하는 실수(순서 누락, 검증 생략, 범위 과대화)를 줄인다.
- 팀 단위 협업에서 의사결정 기준과 보고 형식을 통일한다.

## 핵심 포인트
- 피드백을 먼저 정확히 이해하고 사실 관계를 검증한다.
- 유효한 지적은 수용하고 수정 계획/결과를 명확히 답한다.
- 불명확하거나 오해 소지가 있으면 근거와 함께 정중히 재질문한다.

## 팀 적용 가이드
- PR/리뷰/디버깅/계획 수립 시 이 문서의 체크리스트를 먼저 확인한다.
- 개인 선호보다 스킬의 게이트 조건(승인, 검증, 옵션 제시)을 우선 적용한다.
- 예외가 필요하면 근거를 남기고 팀에 공유한다.

## Example usage
```text
"리뷰 코멘트가 여러 개 달렸어. receiving-code-review 원칙으로 코멘트별 대응안을 작성해줘."
```
