# JIRA Agent 서비스 개발 예정
1. 동료 3명이 JIRA Agent 서비스 개발 계획을 수립하는 중입니다.
2. AI(LLM)를 이용하여 JIRA 를 손쉽게 생성하는 서비스 입니다.
3. Use case
  1) 회의록 내용을 입력하면 요약해서 A/I(Action Item) 2~3개로 요약하여 JIRA Task를 생성
  2) 필요에 따라 JIRA sub-task 까지 세분화해서 생성
  3) 긴급 수명 업무 내용을 요약해서 A/I(Action Item) 2~3개로 요약하여 JIRA Task를 생성

4. 위 서비스는 웹 기반의 Frontend 에서 사용자가 특정 내용을 입력하면 JIRA 티켓을 생성하는 서비스 입니다.
5. 새로운 팀(240명)의 그라운드룰이 JIRA를 통해서 업무를 관리하는 것입니다. 많은 개발자가 JIRA 생성을 귀찮고 불편하다고 생각합니다.

# 위 내용이 Phase1 기본 요구사항입니다.
# 위 서비스가 활성화되고 사용자의 반응이 좋으면 Phase 2 확장 가능
1. 팀 프로젝트의 Epic 과 Story JIRA를 읽고 파악하여, Task 생성시에 자동으로 연결하는 기능
2. JIRA comment가 업데이트 되면, assigner에게 메일로 noti
3. Conflence MCP와 연동하여 부가 서비스 제공
