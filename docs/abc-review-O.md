# abc-review-O

## 배경과 목표
- 내부 PC에서 Claude Code 사용 불가(보안 프록시) 환경에서 외부 PC로 작업하고, 내부 PC에서 결과를 공유하는 운영 체계를 만든다.
- Jira 주간보고와 Confluence 개발 Tip을 동시에 관리해 업무 공유 신뢰도를 높인다.

## 운영 전략 요약
- 외부 PC에서 작업 후 내부 PC에서 `git pull`로 반영한다.
- Jira는 진행 상태와 리스크 중심, Confluence는 재사용 가능한 개발 Tip 중심으로 작성한다.
- copy & paste 기반으로 빠르게 기록하고, 형식 통일로 품질을 유지한다.

## 실무 체크리스트
### 작업 전 준비
- 이슈 키/목표/범위를 명확히 정의한다.
- 결과물이 Jira 보고 대상인지, Confluence Tip 공유 대상인지 구분한다.

### 외부 PC 작업
- 변경 요약과 핵심 의사결정을 기록한다.
- 재사용 가능한 Tip이 있는지 판별한다.

### 내부 PC 반영
- `git pull` 후 변경 사항을 검증한다.
- Jira/Confluence 템플릿에 맞춰 복붙 형태로 기록한다.

## 개선 전략 제안
- `make-jira`/`make-confluence` 스킬 설계
  - 입력 필드: 이슈 키, 담당자, 일정, 요약, 변경 내용, 리스크/이슈, 검증, 참고 링크
  - 출력 포맷: Jira/Confluence 각각의 복붙 템플릿 자동 생성
- Claude Code 작업 로그 구조화
  - 태그 규칙 예시: [결정], [변경], [리스크], [검증], [링크]

## 운영 팁
- Jira는 주간보고 기준으로 “진척과 리스크”만 유지하고 상세는 Confluence에 집중한다.
- Confluence는 “재사용 가능한 패턴”을 강조해 팀 내 확산 효과를 높인다.
- 리뷰/승인은 Jira에 링크로 연결해 추적성을 확보한다.
