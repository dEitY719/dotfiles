# Frontend Design Skill

**고급 UI/웹 컴포넌트 구축** - "AI 스럽지 않은" 독특한 디자인의 웹앱을 제작합니다.

---

## 🎯 이 스킬이 뭔가요?

대시보드, 랜딩대시보드, 랜딩페이지, React 컴포넌트 같은 웹앱을 만들 때, **진정한 디자인**을 합니다. 단순히 기능만 작동하는 것이 아니라,
**사용자가 기억할 만큼 아름답고 의도적인** 인터페이스를 만듭니다.

마치 전문 디자이너가 설계한 것처럼, 모든 색상, 폰트, 애니메이션, 레이아웃이 그 목적에 맞게 최적화됩니다.

---

## 🔄 어떻게 작동하나요?

### 4단계 프로세스

```text
목적 파악 → 디자인 철학 선택 → 코드 구현 → 최적화
```text

#### 1. 목적 파악

- **문제**: 이 인터페이스가 해결할 문제는?

- **대상**: 누가 사용할 것인가?

- **톤**: 기업적? 장난스러운? 미니멀? 최대한 개성 있게?

#### 2. 디자인 철학 선택

극단적인 선택 (중간은 없음):

- **Brutalist Minimalism**: 대담함과 단순함

- **Maximalist Chaos**: 풍부한 색상과 디테일

- **Retro-Futuristic**: 과거와 미래의 혼합

- **Organic & Natural**: 자연스러운 곡선과 따뜻함

- **Editorial/Magazine**: 잡지 스타일의 우아함

- 등등...

**중요**: 회색빛의 중간은 피합니다!

#### 3. 코드 구현

HTML/CSS/JS 또는 React로 실제 코드 작성:

- **Typography**: 일반적이지 않은 흥미로운 폰트 조합

- **Color**: 대담한 팔레트, 강한 대비

- **Motion**: CSS 애니메이션, 마이크로인터랙션

- **Layout**: 비대칭, 격자 깨기, 예상 밖의 배치

#### 4. 최적화

- 접근성 확인

- 성능 검증

- 다양한 화면 크기 테스트

## 📊 Output (생산물)

### 완성된 웹앱

```text
┌────────────────────────────────┐
│  [독특한 헤더 디자인]           │
│  특이한 폰트, 대담한 색상       │
│                                │
│  [의도적인 레이아웃]            │
│  비대칭, 여백 활용              │
│                                │
│  [인터랙티브 요소]              │
│  호버 효과, 애니메이션          │
│                                │
│  [일관된 디자인 언어]           │
│  모든 요소가 조화               │
└────────────────────────────────┘
```

**특징**:

- 완전히 기능함

- 생산 품질 (프로덕션 레벨)

- 독특하고 기억에 남는 디자인

- 반응형 웹 (responsive)

## 💡 실제 사용 예시

### 예시 1: 대시보드 웹앱

#### 요청 (대시)

> "팀의 프로젝트 대시보드를 만들어줘. 세련되고 기억에 남도록."

#### 디자인 철학 (대시)

```text
"Modern Industrial" - 산업적 우아함

강한 그리드, 금속 같은 회색과 따뜻한 주황색의 대비,
작은 애니메이션으로 인터랙티브함을 표현...
```

#### 결과

- 칼 같은 정렬, 강력한 타이포그래피

- 데이터 시각화는 주황색 강조

- 호버 효과로 상호작용성 표현

- 로딩 애니메이션은 세련되고 부드러움

### 예시 2: 랜딩페이지

#### 요청

> "온라인 코스 플랫폼 랜딩페이지. 따뜻하고 친근해야 해."

#### 디자인 철학 (랜딩)

```text
"Warm Mentorship" - 따뜻한 멘토십

둥근 모양, 유기적 배열, 자연 색상 (크림, 녹색, 따뜻한 갈색),
부드러운 타이포그래피, 사진은 따뜻하고 인간적으로...
```

#### 결과 (랜딩)

- 곡선적인 배치, 부드러운 트랜지션

- 따뜻한 색상 팔레트

- 인간적인 이미지와 텍스트의 혼합

- 스크롤 애니메이션으로 생동감

## 🎯 특징

| 특징 | 설명 |
| --- | --- |

| **디자인 철학** | 명확한 방향과 의도 |
| **독특한 폰트** | 일반적이지 않은 조합 |
| **대담한 색상** | 밝고 강한 팔레트 |
| **모션 디자인** | CSS 애니메이션, 마이크로인터랙션 |
| **비대칭 레이아웃** | 격자 깨기, 예상 밖의 배치 |
| **완전 기능** | 단순한 목업이 아닌 실제 작동 |
| **반응형** | 모든 화면 크기에 최적화 |
| **No AI Slop** | 자극적이거나 보편적인 디자인 거부 |

## 🚀 시작하기

### 사용자 입장 (Claude 사용할 때)

```text
"[설명]을 웹앱으로 만들어줘. 아름답고 독특하게."
```text

또는

```text
"대시보드를 만드는데, [톤]이 강조되어야 해. 기억에 남는 디자인으로."
```

### 기술 선택

- **간단한 페이지**: HTML + CSS + Vanilla JS

- **복잡한 상호작용**: React + Tailwind CSS

- **고급 애니메이션**: Framer Motion, GSAP

### 결과물 활용

1. **링크 공유**: 바로 브라우저에서 보기
2. **코드 복사**: 자신의 프로젝트에 통합
3. **배포**: Vercel, Netlify 등으로 즉시 배포
4. **개선**: "헤더를 더 크게 해줄래?" 하면 수정

## 📚 스킬의 핵심 철학

> **"좋은 디자인은 의도에서 나온다"**

- 모든 색상 선택은 의도적

- 모든 폰트 조합은 목적이 있음

- 애니메이션은 단순 치장이 아니라 경험의 일부

- 사용자가 "이거 멋진데?"라고 말할 정도로

## ❓ FAQ

**Q: 일반적인 디자인은 안 되나요?**

A: 우리는 일반적인 것을 거부합니다! 가능한 대담하고 의도적이어야 합니다.

**Q: 접근성은?**

A: 아름다움과 접근성은 양립합니다. 색상 대비, 폰트 크기 등 모두 검증합니다.

**Q: 반응형은?**

A: 필수입니다! 모바일, 태블릿, 데스크톱 모두 최적화됩니다.

**Q: 느린 애니메이션이 싫으면?**

A: 설정으로 조절 가능합니다. "애니메이션을 더 빠르게"하면 수정합니다.
