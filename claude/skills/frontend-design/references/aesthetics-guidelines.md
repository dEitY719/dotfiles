# Aesthetics Guidelines — implementation detail

Apply these rules after choosing an explicit visual direction.

## Typography
- Use distinctive, context-appropriate font pairings.
- Avoid generic defaults (Arial, Roboto, Inter, system stack-only solutions).
- Pair a characterful display font with a readable body font.

## Color and Theme
- Define CSS variables/tokens for palette consistency.
- Prefer decisive dominant colors with intentional accents.
- Avoid timid, evenly distributed palettes that blur hierarchy.

## Motion
- Use motion to support hierarchy and delight, not decoration noise.
- Prefer a few high-impact sequences (e.g., staged load reveal) over constant micro-motion.
- Keep transitions purposeful for hover, scroll, and state changes.

## Spatial Composition
- Use intentional composition: asymmetry, overlap, diagonal flow, or strict rhythm.
- Control density: either generous negative space or deliberate compactness.
- Break grid only when it reinforces the concept.

## Background and Surface Detail
- Build atmosphere with depth: gradients, textures, patterns, or layered transparencies.
- Use shadows/borders/grain/cursors only when they support the chosen tone.

## Anti-Patterns
- Do not default to purple-on-white gradient clichés.
- Do not reuse cookie-cutter landing page sections without contextual adaptation.
- Do not converge on the same font/theme combination across unrelated tasks.
