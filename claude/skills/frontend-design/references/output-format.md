# Output Format — final response contract

Use this structure in the final response.

## Required Sections
1. Aesthetic Direction
2. Implementation Summary
3. Why This Works
4. Validation Notes
5. Next Steps (optional)

## Section Guidance
- Aesthetic Direction: one-paragraph concept statement (tone + differentiation).
- Implementation Summary: what was built, stack used, and key technical decisions.
- Why This Works: map major visual choices to user/context goals.
- Validation Notes: responsiveness, accessibility basics, functional checks.
- Next Steps: only include when meaningful follow-up exists.

## Example Skeleton
```md
## Aesthetic Direction
<direction statement>

## Implementation Summary
<what was built>

## Why This Works
<decision rationale>

## Validation Notes
- Responsive: <result>
- Accessibility basics: <result>
- Functional behavior: <result>
```
