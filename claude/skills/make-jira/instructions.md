# make-jira: Detailed Implementation Instructions

## Overview

Generate comprehensive weekly Jira reports by parsing work logs, grouping by Jira key, aggregating metrics, and fetching commit details.

## Input Data Source

### Primary: `~/work_log.txt`

Format (auto-generated by post-commit hook + work-log CLI):
```
[YYYY-MM-DD HH:MM:SS] [JIRA-KEY] | type | category | time | source
  └─ Category: CategoryName
```

Example entries:
```
[2026-01-27 17:29:01] [PROJ-999] | main | 0.5h | 7addef2
  └─ Category: Testing
[2026-01-27 18:23:45] [SWINNOTEAM-903] | coordination | Communication | 2.5h | manual
  └─ Category: Communication
[2026-01-27 18:25:31] [SWINNOTEAM-906] | meeting | Communication | 1h | manual
  └─ Category: Communication
```

### Secondary: `git log`

Fetch commit messages for context:
```bash
git log --grep="JIRA-KEY" --format="%H%n%s%n%b%n---" --date=short
```

## Processing Steps

### Step 1: Parse work_log.txt

Extract from each entry:
- **Timestamp**: `[YYYY-MM-DD HH:MM:SS]`
- **Jira Key**: `[SWINNOTEAM-906]`
- **Type**: `coordination|assessment|approval|meeting|main` (post-commit hook uses "main")
- **Category**: `Testing|Infrastructure|Communication|...`
- **Hours**: `0.5h|1h|2.5h|1.5h` (normalize to float)
- **Source**: `commit-hash|manual`

Regex pattern:
```
\[(\d{4}-\d{2}-\d{2} \d{2}:\d{2}:\d{2})\] \[([A-Z][A-Z0-9]*-\d+)\] \| ([^\|]+) \| ([^\|]+) \| ([0-9.]+)h? \| ([a-z0-9]+)
```

### Step 2: Filter by Week

Current week calculation:
```
Today: 2026-01-27 (Monday is ISO week day 1)

ISO Week Start (Monday):
  - If today is Monday: use today
  - Otherwise: go back to nearest Monday

Example: 2026-01-27 = Monday → Week start: 2026-01-26
ISO Week Number: date +%V → 05
Format: 2026-W05
```

Filter entries where:
- `timestamp >= Monday 2026-01-26 00:00:00`
- `timestamp <= Sunday 2026-02-01 23:59:59`

### Step 3: Group by Jira Key

Aggregate:
```
SWINNOTEAM-903:
  - entries: [
      {timestamp, type, category, hours, source},
      ...
    ]
  - total_hours: 2.5
  - categories: {Communication: 2.5}
  - sources: {manual: 2.5}

SWINNOTEAM-906:
  - entries: [...]
  - total_hours: 4.5
  - categories: {Testing: 4.5}
  - sources: {manual: 1.5, "7addef2": 3.0}
```

### Step 4: Fetch Commit Details (Optional)

For entries with commit hash source:
```bash
git show --format="%H%n%s%n%b" <commit-hash> --no-patch
```

Extract:
- Commit subject (title)
- Full message (body)
- If body contains metrics, extract them

### Step 5: Generate Summary

Create 2-3 key accomplishments from aggregated data:
- Use metrics from commit messages
- Group related Jira keys by category
- Order by impact/priority

### Step 6: Build Report Structure

Template:
```markdown
[주간보고] YYYY-W## (Start Date ~ End Date)

요약
- Achievement 1 with metrics
- Achievement 2 with metrics

완료 (Done)
- JIRA-KEY-1: Feature Title
  * Specific result 1
  * Specific result 2
- JIRA-KEY-2: Another Feature
  * Result 1

Work Log Summary
- JIRA-KEY-1: Total hours (Category: Hours)
- JIRA-KEY-2: Total hours (Category: Hours)
- Total: X.Xh

추가 참고
- Link to rca-knowledge/docs/analysis/ if applicable
- Link to git commits
- Next week focus areas
```

### Step 7: Save Output

Location: `rca-knowledge/docs/jira-records/YYYY-W##-report.md`

Naming:
- Directory: Must exist: `rca-knowledge/docs/jira-records/`
- Filename: `2026-W05-report.md` (ISO week format)
- Backup: If file exists, append timestamp: `2026-W05-report-20260127-1830.md.backup`
