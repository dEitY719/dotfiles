# test — 본문 템플릿

테스트 추가/변경 이슈에 사용한다 (제품 동작 변경 없음).

## 타이틀

```
test[(<scope>)]: <한 줄 요약>
```

예) `test(bats): setup.sh mocks 를 TEST_TEMP_HOME 으로 격리`

## 본문 골격

```markdown
## TL;DR
<1~3줄 — 어떤 갭을 어떤 시나리오로 메우는가>

## 테스트 갭 (Coverage Gap)
<현재 커버되지 않는 동작·조건. 사고/리뷰에서 드러난 사례라면 인용>

## 추가 시나리오
- <시나리오 1: 입력 → 기대 결과>
- <시나리오 2: ...>

## 셋업 / 더블
<픽스처·목·임시 디렉토리·외부 의존 가짜화 전략>

## 검증
<새 테스트가 의도한 회귀를 실제로 잡는지 확인하는 방법
(예: 가짜로 회귀 도입 → 실패 확인 → 원복)>

## References
- 관련 파일·이슈/PR
- 회귀가 발생했던 사고·리뷰 코멘트 (있으면)
```

## 작성 노트

- 제품 동작 변경이 섞이면 feat / fix / refactor 다. 본 템플릿은
  테스트 자체의 추가·강화·리팩토링에만 사용.
- 새 테스트는 도입 시 의도한 회귀를 실제로 잡는지 한 번 검증한다 —
  통과만 시키는 테스트는 가치가 적다.
