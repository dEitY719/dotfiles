# docs — 본문 템플릿

문서 자체의 변경 이슈에 사용한다 (코드 변경 없음).

## 타이틀

```
docs[(<scope>)]: <한 줄 요약>
```

예) `docs(claude): AGENTS.md 100줄 제한 명문화`

## 본문 골격

```markdown
## TL;DR
<1~3줄 — 어떤 문서를 어떻게 바꾸고 왜>

## 대상 문서
- `path/to/doc.md`

## 변경 내용
<섹션 추가·삭제·재구성·표현 정정 등>

## 동기
<왜 지금 변경하는가 — 사용자 혼란·신규 결정·외부 변경 반영 등>

## References
- 관련 이슈/PR
- 외부 표준·소스 (있으면)
```

## 작성 노트

- 코드 변경이 함께 필요하면 docs 가 아니라 feat / fix / refactor 다.
- 다국어 문서를 동시에 갱신할 때는 "변경 내용" 에 언어별 섹션을 둔다.
