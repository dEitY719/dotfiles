# perf — 본문 템플릿

성능 개선 이슈에 사용한다.

## 타이틀

```
perf[(<scope>)]: <한 줄 요약>
```

예) `perf(setup): 초기 부팅 시간 1.4s → 0.3s 로 단축`

## 본문 골격

```markdown
## TL;DR
<1~3줄 — 어떤 지표를 어디까지 개선하려는가>

## 현황 측정 (Baseline)
<현재 측정값. 측정 환경·방법·반복 횟수 명시>

## 목표 (Target)
<수치 목표·SLO. 측정 가능해야 함>

## 분석 (Bottleneck)
<프로파일·트레이스·핫스팟. 도구 산출물 인용>

## 최적화 안
<제안 변경. 대안과 트레이드오프>

## 검증 (Before/After 측정)
<같은 환경·방법으로 재측정한 결과>

## References
- 관련 파일·도구·이슈
- 측정 데이터·로그
```

## 작성 노트

- Baseline 과 검증의 측정 환경은 동일해야 한다 — 측정 조건 차이는
  결론을 무효화한다.
- 회귀 방지 — 가능하면 베이스라인 측정을 자동화 테스트로 고정.
