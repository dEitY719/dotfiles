# /debug — 세션 디버그 로그 분석기

Claude Code 내장(built-in) 스킬. 플러그인/마켓플레이스가 아닌 Claude Code 바이너리에 포함되어 있어 파일시스템에 별도의 `SKILL.md`가 존재하지 않는다. 현재 세션의 디버그 로그를 읽어 오류·경고·실패 패턴을 분석하고, 구체적인 수정 방향을 제시한다.

## 동작 요약

| Phase | 이름 | 핵심 동작 |
| ----- | ---- | --------- |
| 1 | 디버그 로깅 상태 확인 | 스킬 호출 전 디버그 로깅이 켜져 있었는지 판별 |
| 2 | 로그 파일 읽기 | 세션 로그 파일 끝에서 최대 64KB 읽기, 마지막 20줄 추출 |
| 3 | 프롬프트 주입 | 수집된 정보를 컨텍스트에 삽입 — 모델 자동 응답 없음 |
| 4 | 사용자 후속 입력 대기 | 사용자가 다음 메시지를 보내면 Claude가 로그를 분석하고 진단 결과 출력 |

## Phase 1: 디버그 로깅 상태 확인

스킬이 호출되는 순간, 이미 디버그 로깅이 활성화되어 있었는지 확인한다(`m8L()` 호출).

### 로깅이 꺼져 있던 경우 (가장 흔한 케이스)

프롬프트 상단에 아래 경고 블록이 추가된다.

```
## Debug Logging Just Enabled
Debug logging was OFF for this session until now.
Nothing prior to this /debug invocation was captured.
```

Claude는 이 정보를 바탕으로 사용자에게 다음을 안내한다.

| 안내 항목 | 내용 |
| --------- | ---- |
| 로그 경로 공지 | 디버그 로그가 저장되는 경로 알림 |
| 재현 요청 | 문제를 다시 재현하여 로그를 생성하도록 요청 |
| 시작 시점 캡처 방법 | 시작부터 캡처가 필요하면 `claude --debug`로 재실행 권고 |

### 로깅이 이미 켜져 있던 경우

경고 블록 없이 곧바로 Phase 2로 진행한다. 스킬 호출 이전의 로그도 파일에 존재하므로 전체 분석이 가능하다.

## Phase 2: 로그 파일 읽기

### 파일 크기 처리

| 조건 | 처리 방식 |
| ---- | --------- |
| 파일이 64KB 이하 | 전체 내용 읽기 |
| 파일이 64KB 초과 | 파일 끝에서 64KB(`dK_ = 65536`)만큼 읽기 |
| 파일이 존재하지 않음 (ENOENT) | "No debug log exists yet — logging was just enabled." 메시지 표시 |

### 마지막 20줄 추출

파일에서 마지막 20줄(`Vd$ = 20`)을 추출하여 프롬프트에 포함한다. 이 줄들은 로그 포맷을 Claude에게 보여주는 역할을 하며, 이후 `[ERROR]`·`[WARN]` 필터링의 기준이 된다.

### 로그 크기 정보

프롬프트에 로그 파일의 전체 크기 정보가 함께 기재되어, Claude가 파일에 더 많은 내용이 있음을 인지하고 필요 시 전체 파일을 grep하도록 유도한다.

## Phase 3: 프롬프트 주입

### 주입되는 컨텍스트 블록

스킬은 다음 요소들을 조합하여 단일 프롬프트를 구성하고 컨텍스트에 삽입한다.

| 블록 | 내용 |
| ---- | ---- |
| 로깅 상태 알림 | 로깅이 방금 활성화되었는지 여부 |
| 로그 파일 경로 | `IDH()`가 반환하는 세션 로그 경로 |
| 로그 크기 정보 | 전체 바이트 수 또는 "파일 없음" |
| 마지막 20줄 | 코드 블록으로 포함된 최근 로그 라인 |
| 문제 설명 | 사용자가 인수로 입력한 설명, 없으면 기본값 |
| 설정 파일 경로 | user / project / local 설정 파일 경로 세 가지 |
| 분석 지시사항 | Claude가 따라야 할 5단계 분석 절차 |

### 문제 설명 기본값

사용자가 `/debug` 만 입력하고 설명을 생략한 경우:

```
The user did not describe a specific issue.
Read the debug log and summarize any errors, warnings, or notable issues.
```

### `disableModelInvocation: true` 동작

이 스킬은 `disableModelInvocation: true`로 등록되어 있다. 스킬이 프롬프트를 컨텍스트에 삽입한 후 **Claude의 자동 응답이 트리거되지 않는다.** 사용자가 다음 메시지를 전송해야 Claude가 분석을 시작한다.

## Phase 4: 사용자 후속 입력 대기 및 분석

사용자가 다음 메시지를 보내면 Claude는 주입된 컨텍스트를 바탕으로 아래 5단계 절차를 따른다.

| 단계 | 작업 | 설명 |
| ---- | ---- | ---- |
| 1 | 문제 설명 검토 | 사용자가 기술한 증상과 로그를 대조 |
| 2 | 오류·경고 탐색 | 마지막 20줄을 참고하여 전체 파일에서 `[ERROR]`·`[WARN]`, 스택 트레이스, 실패 패턴 grep |
| 3 | 서브에이전트 활용 고려 | `claude-code-guide` 서브에이전트(`MVA`)를 실행하여 관련 Claude Code 기능 이해 |
| 4 | 진단 결과 설명 | 발견한 내용을 평문으로 설명 |
| 5 | 수정 방안 제시 | 구체적인 수정 방법 또는 다음 단계 제안 |

### 설정 파일 경로 참조

프롬프트에는 세 가지 설정 파일 경로가 명시된다.

| 범위 | 설명 |
| ---- | ---- |
| user | 전역 사용자 설정 |
| project | 프로젝트 루트 설정 |
| local | 로컬(gitignore) 설정 |

Claude가 설정 관련 오류를 진단할 때 이 경로를 참조하여 파일을 직접 읽을 수 있다.

## 허용 도구

스킬 등록 시 `allowedTools: ["Read", "Grep", "Glob"]`로 제한되어 있다. 쓰기 도구(Write, Edit, Bash 등)는 허용되지 않으므로 진단 작업에만 사용 가능하다.

## 특징

- **사용자 전용 호출**: `disableModelInvocation: true`로 설정되어 있어 Claude가 자동으로 트리거할 수 없고, 사용자가 `/debug`로 직접 호출해야 한다.
- **지연 활성화 대응**: 디버그 로깅이 꺼진 상태에서 호출되어도 동작한다. 다만 호출 이전 로그는 없으므로 재현을 유도하는 메시지를 먼저 출력한다.
- **64KB 테일 읽기**: 로그가 아무리 커도 파일 끝에서 64KB만 읽어 성능 저하 없이 최신 정보를 확보한다.
- **20줄 앵커**: 마지막 20줄을 프롬프트에 직접 포함하여 Claude가 로그 포맷을 즉시 파악하고 전체 파일 grep 방향을 잡을 수 있게 한다.
- **읽기 전용 도구 제한**: 진단 도구(Read, Grep, Glob)만 허용하여 의도치 않은 파일 수정을 방지한다.
- **`claude-code-guide` 연계**: 스킬 자체가 알 수 없는 Claude Code 내부 동작을 `claude-code-guide` 서브에이전트에 위임하여 진단 깊이를 확장한다.
- **설정 파일 경로 자동 제공**: user·project·local 세 가지 설정 경로를 프롬프트에 포함하여 설정 오류 진단 시 추가 탐색 없이 바로 파일을 읽을 수 있다.
