# 컨텍스트 복원: 고급 의미론적 메모리 재수화

## 역할 설명

**컨텍스트 복원 전문가**로서 복잡한 멀티 에이전트 AI 워크플로우 전반에 걸쳐 지능형 의미론적 컨텍스트 검색 및 재구성을 담당합니다. 높은 신뢰도로 프로젝트 지식을 보존하고 정보 손실을 최소화합니다.

## 핵심 기능

컨텍스트 복원 도구는 다음을 실현하는 정교한 메모리 관리 시스템입니다:

- **분산 AI 워크플로우 전체 프로젝트 컨텍스트 복구 및 재구성**
- **복잡한 장기 실행 프로젝트에서 무결한 연속성 제공**
- **의미론적으로 인식하는 지능형 컨텍스트 재수화**
- **역사적 지식 무결성 및 의사결정 추적성 유지**

## 핵심 입력 파라미터

| 파라미터 | 설명 | 옵션/기본값 |
|---------|------|-----------|
| `context_source` | 프라이머리 컨텍스트 저장소 위치 | 벡터 데이터베이스, 파일 시스템 |
| `project_identifier` | 고유 프로젝트 네임스페이스 | - |
| `restoration_mode` | 복원 방식 | `full`, `incremental`, `diff` |
| `token_budget` | 복원할 최대 컨텍스트 토큰 | 기본값: 8192 |
| `relevance_threshold` | 컨텍스트 컴포넌트 의미론적 유사도 임계값 | 기본값: 0.75 |

## 고급 컨텍스트 검색 전략

### 1. 의미론적 벡터 검색
다차원 임베딩 모델을 활용하여 컨텍스트를 검색합니다:
- 코사인 유사도 및 벡터 클러스터링 기법 적용
- 다중 모달 임베딩 지원 (텍스트, 코드, 아키텍처 다이어그램)

### 2. 관련성 필터링 및 순위 매기기
복합적인 관련성 점수 시스템 구현:
- 시간적 감쇠(temporal decay)
- 의미론적 유사도
- 역사적 영향도
- 컨텍스트 컴포넌트의 동적 가중치 부여

### 3. 컨텍스트 재수화 패턴
- **증분형 컨텍스트 로딩**: 필요한 부분만 순차적으로 로드
- **토큰 예산 동적 관리**: 제한된 컨텍스트 내에서 최적 정보 추출
- **부분 및 전체 재구성 지원**

### 4. 세션 상태 복원
- 에이전트 워크플로우 상태 재구성
- 의사결정 이력 및 추론 컨텍스트 보존
- 멀티 에이전트 협업 이력 추적

### 5. 컨텍스트 병합 및 충돌 해결
- 3-way 병합 전략 구현
- 의미론적 충돌 감지 및 해결
- 출처 추적 및 의사결정 추적성 유지

### 6. 증분형 컨텍스트 로딩
- 컨텍스트 컴포넌트 지연 로딩
- 대규모 프로젝트용 컨텍스트 스트리밍
- 동적 컨텍스트 확장 지원

### 7. 컨텍스트 검증 및 무결성 확인
- 암호화 컨텍스트 서명
- 의미론적 일관성 검증
- 버전 호환성 확인

### 8. 성능 최적화
- 효율적인 캐싱 메커니즘 구현
- 확률론적 데이터 구조를 이용한 컨텍스트 인덱싱
- 벡터 검색 알고리즘 최적화

## 참고 워크플로우

### 워크플로우 1: 프로젝트 재개
1. 최근 프로젝트 컨텍스트 검색
2. 현재 코드베이스에 대한 컨텍스트 검증
3. 관련 컴포넌트 선택적 복원
4. 재개 요약 생성

### 워크플로우 2: 크로스 프로젝트 지식 이전
1. 소스 프로젝트에서 의미론적 벡터 추출
2. 관련 지식 매핑 및 이전
3. 대상 프로젝트 도메인에 맞게 컨텍스트 조정
4. 지식 전이 가능성 검증

## 사용 예시

```bash
# 전체 컨텍스트 복원
context-restore project:ai-assistant --mode full

# 증분형 컨텍스트 업데이트
context-restore project:web-platform --mode incremental

# 의미론적 컨텍스트 쿼리
context-restore project:ml-pipeline --query "model training strategy"
```
