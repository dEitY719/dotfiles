# Prompt Optimization 요약

## 목적
- 단순한 요청을 **정확도·안전성·일관성·비용 효율**이 높은 “프로덕션용 프롬프트”로 변환하는 절차와 템플릿을 제공한다.
- 토큰 최적화 및 실패 모드/엣지 케이스 대응을 통해 **환각 감소, 비용 절감, 성능 개선**을 목표로 한다.

## 핵심 워크플로(단계별)
1. **현재 프롬프트 분석**
   - 명확성(점수화), 모호성 지점, 구조(섹션 경계/흐름), 모델 적합성(토큰 효율 포함), 성능(성공률·실패 모드·엣지 케이스)을 평가
   - 목표/제약, 출력 형식, 명시 vs 암묵 요구사항, 컨텍스트 의존성/가변 요소를 분해
2. **추론 강화(Reasoning Enhancement)**
   - 문제를 단계로 쪼개는 “단계적 추론” 패턴, 간단 지시문에 단계적 접근 문구를 추가하는 방식, 다중 해법을 탐색·평가하는 트리 기반 사고 패턴을 제시
3. **Few-shot 학습 적용**
   - 단순 케이스/엣지 케이스/오답-정답 대비 예시를 전략적으로 배치해 출력 준수율을 높이는 템플릿 제공
4. **Constitutional AI(자기 점검) 패턴**
   - 초안 생성 후 **정확성·안전성·품질** 원칙으로 자체 검토하고 최종안을 개선하는 루프 구조 제시
5. **모델별 최적화 템플릿**
   - GPT 계열: `CONTEXT/OBJECTIVE/INSTRUCTIONS/OUTPUT FORMAT/EXAMPLES`처럼 섹션을 강하게 구조화하고 JSON 등 출력 스키마를 명시
   - Claude 계열: `<context>/<task>/<thinking>/<output_format>` 같은 태그 기반 구조 예시
   - Gemini 계열: 시스템 컨텍스트·목표·프로세스·출력 구조·품질 제약을 항목화
6. **RAG 통합**
   - 검색 문서(컨텍스트)와 사용자 질문을 함께 제공하고, 관련성 판단→합성(출처 표기)→커버리지/공백 명시→응답의 절차를 정의
7. **평가 프레임워크**
   - 20개 테스트 케이스(일반/엣지/적대/범위 밖)와 성공률·품질·효율·안전 지표 제시
   - LLM-as-Judge 평가 프롬프트로 작업 완수도/정확성/구조/형식/안전을 점수화하고 Accept/Revise/Reject 권고
8. **프로덕션 배포 고려**
   - 프롬프트 버저닝, 카나리/스테이징 롤아웃, 롤백 임계치, 모니터링 기간 등 운영 전략 예시
   - 정보 부족/요구사항 충돌/능력 한계/안전 이슈에 대한 에러 핸들링 및 “부분 해법+경계+다음 단계”의 점진적 저하 전략

## 제공되는 산출물(출력 형식)
- **Optimized Prompt**: 강화 기법을 반영한 최종 프롬프트(마크다운 형태)
- **Optimization Report (YAML)**: 기존 평가, 적용 기법과 기대 효과, 성능/효율/안전 개선 추정, 테스트 권고, 배포 파라미터 및 다음 단계

## 사용 가이드라인(요지)
- 최적화된 프롬프트를 그대로 사용하고, 권장 파라미터를 적용하며, 테스트 케이스로 검증 후 배포/모니터링/반복 개선한다.
