# LLM 평가(LLM Evaluation) 요약

## 목적
- LLM 애플리케이션의 품질을 **정량(자동화 지표)** 과 **정성(사람 평가/판정)** 으로 체계적으로 측정하고, **비교·회귀 감지·개선 검증**까지 포함하는 평가 체계를 구축합니다.

## 사용 시점(When to Use)
- LLM 성능을 **지속적으로 측정**하고 기준선(baseline)을 관리할 때
- **모델/프롬프트 간 비교**가 필요할 때
- 배포 전 **성능 회귀(regression)** 를 조기에 탐지할 때
- 프롬프트 변경/개선이 **실제로 향상인지 검증**할 때
- 운영 환경에서 품질에 대한 **신뢰도**를 확보할 때
- 예상치 못한 출력(환각/오답 등)을 **디버깅**할 때

## 핵심 평가 유형(Core Evaluation Types)

### 1) 자동화 지표(Automated Metrics)
- 빠르고 반복 가능하며 대규모로 확장 가능한 **점수 기반 평가**입니다.
- **텍스트 생성**
  - `BLEU`: N-그램 중복 기반(번역 등)
  - `ROUGE`: 재현율 중심(요약 등)
  - `METEOR`: 의미 유사도 고려
  - `BERTScore`: 임베딩 기반 유사도
  - `Perplexity`: 언어모델의 예측 확신도/난이도
- **분류(Classification)**
  - `Accuracy`, `Precision/Recall/F1`, `Confusion Matrix`, `AUC-ROC`
- **검색/근거기반 생성(RAG)**
  - `MRR`, `NDCG`, `Precision@K`, `Recall@K`

### 2) 사람 평가(Human Evaluation)
- 자동화가 어려운 품질 요소를 사람이 직접 평가합니다.
- 주요 평가 차원:
  - 정확성(사실성), 일관성(논리 흐름), 관련성, 유창성(문장 품질), 안전성, 유용성

### 3) LLM-as-Judge(LLM 판정)
- 더 강한 LLM을 “심사위원”으로 사용해 출력 품질을 평가합니다.
- 방식:
  - Pointwise(단일 응답 채점), Pairwise(두 응답 비교), Reference-based(정답 기준 비교), Reference-free(정답 없이 판단)

## 빠른 시작(Quick Start) 개념
- `EvaluationSuite`에 여러 `Metric`을 구성하고, `input/expected/context`를 포함한 테스트 케이스로 평가를 실행합니다.
- 기본 지표(정확도, BLEU, BERTScore) + 커스텀 지표(예: groundedness)를 함께 묶어 종합 점수를 얻는 패턴을 제시합니다.

## 구현 예시에서 다루는 것(요지)
- `BLEU/ROUGE/BERTScore` 계산 예시
- 커스텀 지표 예시:
  - `groundedness`: 응답이 주어진 컨텍스트에 근거하는지(NLI 활용)
  - `toxicity`: 유해성 점수(detoxify 등)
  - `factuality`: 지식베이스 기반 사실 검증(구현은 환경 의존)
- LLM-as-Judge 패턴:
  - 단일 응답을 정확성/유용성/명확성 등으로 JSON 채점
  - A/B 응답을 비교해 승자·근거·신뢰도를 JSON으로 반환

## 사람 평가 프레임워크(Human Evaluation Frameworks)
- 어노테이션 폼(평가 항목/스케일/이슈 플래그/피드백)을 구조화하는 예시 제공
- 평가자 간 일치도(Inter-Rater Agreement): `Cohen’s kappa`로 합의 수준을 계산하는 접근을 소개

## A/B 테스트
- 두 변형(A/B)의 점수를 수집하고, 통계적 검정(t-test)과 효과크기(Cohen’s d)로 차이를 분석합니다.
- 평균, 차이, 상대 개선률, p-value, 유의성 여부, 효과크기 해석, 승자 판정을 반환하는 형태를 제시합니다.

## 회귀 테스트(Regression Testing)
- 기준선 대비 새 결과의 **상대 변화율**을 계산해, 임계치(threshold) 이상 하락 시 회귀로 플래그합니다.
- 어떤 지표가 얼마나 떨어졌는지(기준/현재/변화)를 목록화합니다.

## 벤치마킹(Benchmarking)
- 벤치마크 데이터셋을 순회하며 예측을 생성하고, 여러 지표를 계산해 평균/표준편차/최솟값/최댓값으로 집계합니다.

## 리소스(Resources)
- 지표 가이드, 사람 평가 가이드, 벤치마크, A/B 테스트, 회귀 테스트 문서 참조
- 평가 하네스/예시 데이터셋/자동 평가 스크립트 자산 제공을 전제로 함

## 모범 사례(Best Practices)
- 단일 지표에 의존하지 말고 **다중 지표**로 평가
- **대표성 있는 데이터**로 테스트
- 항상 **기준선 대비 비교**
- 통계적으로 엄밀하게 비교
- CI/CD에 **지속 평가** 통합
- 자동 평가 + **사람 검증** 결합
- 실패 사례 **오류 분석**
- 결과를 버전 관리로 **추적**

## 흔한 함정(Common Pitfalls)
- 한 지표만 최적화(다른 품질 희생)
- 표본 수 부족
- 학습 데이터로 평가(오염)
- 분산/불확실성 무시
- 비즈니스 목표와 맞지 않는 지표 선택(지표 불일치)
