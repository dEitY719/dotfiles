# Similarity Search Patterns (유사도 검색 패턴) 요약

프로덕션 환경에서 **벡터 기반 유사도 검색**을 효율적으로 구현하기 위한 핵심 개념, 인덱스 선택 기준, 그리고 대표 벡터 DB별 구현 템플릿을 제공한다.

## 사용 시점 (When to Use)
- **시맨틱 검색** 시스템 구축(임베딩 기반 검색)
- **RAG 리트리벌**(관련 문서/청크 후보 검색)
- **추천 시스템**(유사 아이템/콘텐츠 탐색)
- **지연시간 최적화**(특히 P95/P99)
- **대규모 벡터 스케일링**(수백만~수억 벡터)
- **하이브리드 검색**(키워드 + 시맨틱 결합)

## 핵심 개념 (Core Concepts)

### 1) 거리/유사도 메트릭 (Distance Metrics)
임베딩 특성과 요구사항에 맞춰 메트릭을 선택한다.
- **Cosine(코사인 거리)**: 정규화된 임베딩에 적합(방향 중심)
- **Euclidean/L2(유클리드 거리)**: 원시 임베딩에서 거리 기반 비교에 적합
- **Dot Product(내적)**: 벡터 크기(노름) 영향이 중요할 때 유용
- **Manhattan/L1(맨해튼 거리)**: 희소 벡터 등에서 활용 가능

### 2) 인덱스 유형 (Index Types)
정확도(리콜)와 속도/메모리의 트레이드오프를 기반으로 선택한다.
- **Flat (Exact)**: O(n) 선형 탐색, **100% 리콜**, 소규모 데이터에 적합
- **HNSW (Graph-based)**: 대개 **로그 수준 탐색**, **약 95~99% 리콜**, 중~대규모에 범용적으로 적합
- **IVF+PQ (Quantized)**: 양자화 기반으로 메모리 효율↑, 초대규모에 유리하나 리콜 손실 가능(대략 90~95%)

## 구현 템플릿 (Templates)

### 1) Pinecone 템플릿
- 인덱스가 없으면 생성(차원/메트릭/리전 지정)
- **배치 업서트**로 대량 적재
- `query`로 **top_k 검색 + 메타데이터 포함**
- **오버페치 후 재랭킹**(CrossEncoder 등) 패턴 포함
- ID 삭제 및 **필터 기반 삭제** 지원

### 2) Qdrant 템플릿
- 컬렉션 자동 생성 + **COSINE 거리**
- 선택적으로 **스칼라 양자화(INT8)**로 메모리 효율 강화
- 업서트는 `PointStruct(id, vector, payload)` 형태
- 검색은 `filter`, `score_threshold` 등으로 제어
- 조건 조합을 통한 **복합 필터 검색** 패턴 제공
- (환경에 따라) **Dense/Sparse 하이브리드** 확장 방향을示

### 3) PostgreSQL + pgvector 템플릿
- `vector` 확장 활성화 및 문서 테이블 생성
- `embedding vector(1536)` 컬럼과 **HNSW 인덱스** 생성으로 성능 개선
- 업서트는 `ON CONFLICT`로 갱신 처리
- `<=>` 연산자를 사용해 **유사도 점수 계산 및 정렬**
- JSONB 메타데이터 기반 **필터링**
- 벡터 검색 + 전문 검색(tsvector/ts_rank)을 결합한 **하이브리드 검색** 예시 제공

### 4) Weaviate 템플릿
- 스키마(클래스) 자동 보장, `vectorizer: none`(외부에서 벡터 제공)
- 배치 업서트 + UUID 생성(`uuid5`)
- `near_vector` 기반 벡터 검색 + 추가 필드(distance/id)
- `with_where`로 필터 적용
- BM25(키워드)와 벡터를 결합한 **하이브리드 검색**(`alpha`로 가중치 조절)

## 모범 사례 (Best Practices)

### Do
- **인덱스 선택 최적화**: 대부분의 케이스에 HNSW가 실용적
- **파라미터 튜닝**: `ef_search`, `nprobe` 등으로 리콜/속도 균형 조정
- **하이브리드 검색 도입**: 키워드 검색과 결합해 품질 향상
- **리콜/품질 모니터링**: 최적화 전후 정량 평가
- **사전 필터링 활용**: 가능한 경우 검색 공간 자체를 축소

### Don’t
- **평가 없이 최적화하지 않기**: 측정 후 조정
- **초기부터 과도한 인덱싱 금지**: Flat으로 시작해 필요 시 확장
- **지연시간 무시 금지**: UX는 P99에 좌우됨
- **비용 고려 누락 금지**: 벡터 저장/인덱스 비용이 누적됨
