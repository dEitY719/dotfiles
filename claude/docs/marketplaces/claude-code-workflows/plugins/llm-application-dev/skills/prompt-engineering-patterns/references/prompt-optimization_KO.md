# Prompt Optimization Guide 요약 (한국어)

## 개요
- 프롬프트 성능을 **정량 지표(정확도, 토큰, 지연시간, 성공률 등)**로 측정하고, **반복 개선(Iterative Refinement)** 및 **A/B 테스트**로 검증해 최적화하는 절차를 제시합니다.

## 1) 체계적 개선 프로세스
### 1. 베이스라인(초기 성능) 확립
- 테스트 케이스 묶음을 대상으로 프롬프트를 실행하고 아래 지표를 집계/평균화합니다.
  - `accuracy`(정확도), `avg_tokens`(평균 토큰), `avg_latency`(평균 지연), `success_rate`(유효 응답 비율)

### 2. 반복 개선 워크플로
- 흐름: **초기 프롬프트 → 테스트 → 실패 분석 → 개선 → 재테스트 → 반복**
- `PromptOptimizer` 개념:
  - 매 반복에서 결과를 기록(`history`)하고, 목표(예: 정확도 0.95 이상)에 도달하면 중단
  - 실패 케이스를 분석해 개선안을 생성하고, 가장 효과적인 개선안을 선택해 프롬프트를 갱신

### 3. A/B 테스트 프레임워크
- 두 프롬프트 변형(Variant A/B)을 **무작위 50/50**로 배정해 실행하고 지표(예: 정확도, 지연시간)를 수집합니다.
- 통계적 유의성 검정:
  - `ttest_ind` 등으로 A/B 차이가 **유의미한지(p<0.05)** 평가
  - 평균 개선율, 승자(winner) 등 비교 리포트를 생성

## 2) 최적화 전략(Optimization Strategies)
### 토큰 절감(Token Reduction)
- 불필요한 표현 제거, 지시문 통합, 약어 사용, 군더더기 단어 제거 등으로 프롬프트 길이를 줄입니다.

### 지연시간 절감(Latency Reduction)
- `shorter_prompt`, `streaming`, `caching`, `early_stopping(Stop sequence)` 같은 전략을 각각 적용해 평균 지연시간을 측정하고, **가장 빠른 전략을 선택**합니다.

### 정확도 향상(Accuracy Improvement)
- 실패 유형에 따라 개선 요소를 누적 적용합니다.
  - 형식 오류: 출력 포맷 제약(JSON 강제 등) 및 예시 추가
  - 엣지 케이스: 대표 예시를 프롬프트에 포함
  - 논리 오류: 응답 전 자체 검증 단계 추가
  - 모호성: 지시문을 더 구체화/명료화

## 3) 성능 지표(Performance Metrics)
### 핵심 지표(Core Metrics)
- `accuracy`: 정답 일치율
- `consistency`: 동일 입력에 대해 출력이 얼마나 일관적인지(최빈 응답 비율 기반)
- `token_efficiency`: 프롬프트+응답의 토큰 비용
- `latency_p95`: 지연시간 95퍼센타일

### 자동 평가(Automated Evaluation)
- 각 테스트 케이스를 여러 번(예: 3회) 실행해 **일관성/변동성**까지 반영합니다.
- 정확도는 “여러 번 중 최선(best-of-runs)” 같은 집계 방식도 포함합니다.
- 최종적으로 평균 정확도, 평균 일관성, p95 지연, 평균 토큰, 성공률을 반환합니다.

## 4) 실패 분석(Failure Analysis)
- 실패를 카테고리로 분류: `format_errors`, `factual_errors`, `logic_errors`, `incomplete_responses`, `hallucinations`, `off_topic`
- 분류 결과에 따라 수정안을 우선순위로 제안:
  - 예: 환각(hallucination) 발생 시 “제공된 컨텍스트만 근거로 답하라” 같은 접지(grounding) 지시를 **중요/긴급**으로 추가

## 5) 버전관리 및 롤백(Versioning & Rollback)
- `PromptVersionControl` 개념:
  - 프롬프트를 버전으로 저장(타임스탬프, 지표, 설명, 부모 버전 링크 등 메타데이터 포함)
  - 특정 버전으로 롤백
  - 두 버전 간 diff 및 지표 비교 제공

## 6) 베스트 프랙티스 & 공통 패턴
- 원칙:
  1) 초기 성능 측정, 2) 한 번에 한 가지 변경, 3) 대표성 있는 테스트, 4) 지표 기록, 5) 통계 검정, 6) 변경 이유 문서화, 7) 버전관리/롤백, 8) 운영 환경 모니터링
- 자주 쓰는 개선 패턴:
  - 구조 추가(항목화), 예시 추가, 제약조건 추가(길이/형식), 검증 단계 추가
- 활용 도구 범주:
  - 프롬프트 diff, 자동 테스트 러너, 지표 대시보드, A/B 테스트 프레임워크, 토큰 카운터, 지연시간 프로파일러
