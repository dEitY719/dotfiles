# Few-Shot Learning Guide (요약)

## 개요
- **Few-shot learning**은 프롬프트에 **소수의 입출력 예시**를 함께 제공해, 모델이 원하는 **출력 형식·스타일·도메인 규칙**을 빠르게 따르도록 만드는 기법이다.
- 특히 **정해진 포맷 준수**, **분류/추출/변환**, **도메인 지식 반영**이 필요한 작업에서 효과적이다.

## 예시 선택(Selection) 전략
### 1) 의미적 유사도 기반(Semantic Similarity)
- 입력 질의와 **의미적으로 가장 유사한 예시**를 임베딩(embedding) 유사도로 상위 k개 선택.
- **적합한 작업**: 질의응답, 텍스트 분류, 정보 추출.

### 2) 다양성 샘플링(Diversity Sampling)
- 다양한 패턴과 엣지 케이스를 커버하도록 **서로 다른 군집 중심(클러스터)** 주변 예시를 선택.
- **적합한 작업**: 과제 변동성 시연, 엣지 케이스 처리 강조.

### 3) 난이도 기반(Difficulty-Based, 점진적 구성)
- 예시에 **난이도 점수**를 두고, 쉬운 것부터 어려운 것까지 **점진적으로 난이도 상승**하도록 구성.
- **적합한 작업**: 복잡 추론, 코드 생성 등 단계적 학습이 유리한 과제.

### 4) 오류 패턴 기반(Error-Based, 실패 모드 대응)
- 자주 발생하는 **실패 유형(오류 패턴)**을 직접 겨냥해, “올바른 처리 방식”을 보여주는 예시 포함.
- **적합한 작업**: 실패 패턴이 알려진 작업, 안전·정확성이 중요한 애플리케이션.

## 예시 구성(Best Practices)
### 형식 일관성(Format Consistency)
- 모든 예시는 **동일한 키/구조/문장 형태**로 작성해 모델이 출력 포맷을 혼동하지 않게 한다.

### 입력-출력 정렬(Input-Output Alignment)
- 예시의 출력은 “원하는 작업 결과”를 **직접적으로** 보여줘야 하며, 불필요한 해설로 관계를 모호하게 만들지 않는다.

### 난이도 균형(Complexity Balance)
- 실제 입력 분포를 반영해 **단순/중간/복잡** 사례를 적절히 섞어 일반화를 돕는다.

## 컨텍스트 윈도우 관리(Context Window)
### 토큰 예산 배분(Token Budget Allocation)
- 시스템 프롬프트/예시/사용자 입력/응답에 대한 **토큰 비중을 계획적으로 배분**해, 예시가 과도하게 공간을 점유하지 않도록 한다.

### 동적 예시 절단(Dynamic Example Truncation)
- 토크나이저로 예시 토큰 수를 계산하면서, 설정한 예산 내에서만 예시를 포함해 **최적 개수·길이**를 유지한다.

## 엣지 케이스 처리(Edge Case Handling)
- 빈 입력, 과도하게 긴 입력, 모호한 입력, 유효하지 않은 입력 등 **경계 사례를 예시로 명시**해 모델의 방어적 응답을 유도한다.

## Few-shot 프롬프트 템플릿
- **분류(Classification)**: “라벨 목록 + 예시(텍스트→카테고리) + 질의” 구조로 마지막 카테고리를 채우게 함.
- **추출(Extraction)**: “원문 + 추출 결과(JSON 등 구조화)” 예시를 반복해 동일한 구조로 출력 유도.
- **변환(Transformation)**: “입력→출력 변환 패턴”을 예시로 보여주고 새 입력에 동일 변환 수행.

## 평가 및 최적화(Evaluation & Optimization)
- **예시 품질 지표**: 명확성, 대표성(검증셋 유사도), 난이도, 고유성 등을 점수화해 예시를 개선.
- **A/B 테스트**: 예시 세트 A/B를 동일 테스트 질의에 적용해 정확도 비교 및 개선폭 측정.

## 고급 기법(Advanced Techniques)
- **메타 러닝(선택 모델 학습)**: 질의-예시 쌍의 특징(유사도, 길이, 키워드 중복 등)으로 “성공할 예시”를 예측하는 선택기를 학습.
- **적응형 예시 개수(Adaptive Example Count)**: 과제 난이도/추정 신뢰도에 따라 예시 수를 1개부터 점진적으로 늘리며 필요한 만큼만 사용.

## 흔한 실수(Common Mistakes)
- 예시를 **너무 많이** 넣어 초점을 흐림
- 질의와 **관련 없는 예시** 선택
- **포맷 불일치**로 출력 형식 혼란 유발
- 예시 패턴에 **과적합**(그대로 모사)
- **토큰 한도 무시**로 실제 입력/출력 공간 부족
