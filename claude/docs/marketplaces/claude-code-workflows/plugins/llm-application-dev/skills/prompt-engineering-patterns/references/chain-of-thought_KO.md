## 개요
- Chain-of-Thought(CoT) 프롬프팅은 모델이 **단계별 추론 과정을 생성**하도록 유도해, 복잡한 문제(추론/수학/논리)에서 성능을 크게 개선하는 접근을 설명합니다.

## 핵심 기법(Core Techniques)
- **Zero-Shot CoT**: 질문 뒤에 `Let's think step by step:` 같은 트리거 문구를 붙여 **추론 전개를 유도**합니다.
- **Few-Shot CoT**: 여러 예시(Q/A)에 **명시적 단계별 풀이**를 포함해, 동일한 형식으로 사용자의 질문을 풀도록 유도합니다.
- **Self-Consistency**: 동일 문제에 대해 **여러 개의 추론 경로를 생성**한 뒤, 최종 답을 **다수결(majority vote)**로 선택하고 신뢰도(빈도 기반)를 산출합니다.

## 고급 패턴(Advanced Patterns)
- **Least-to-Most Prompting**: (1) 문제를 하위 문제로 분해 → (2) 순차적으로 풀이하며 컨텍스트 누적 → (3) 결과를 통합해 최종 답을 도출합니다.
- **Tree-of-Thought(ToT)**: 매 단계에서 **여러 분기(thought branches)**를 생성하고, 각 경로를 점수화(정확성/해결 가능성/논리 일관성 등)해 **가장 유망한 경로**를 선택합니다.
- **Verification Step**: (1) 풀이 생성 → (2) 별도 프롬프트로 단계 오류·산술 검증·타당성 점검 → (3) 오류 감지 시 수정 풀이를 재생성합니다.

## 도메인별 CoT 템플릿(Domain-Specific)
- **수학(Math)**: 알려진 값/목표 변수/공식 선택/대입/계산/검증을 단계화한 템플릿을 제공합니다.
- **코드 디버깅(Code Debugging)**: 에러 메시지 해석 → 문제 라인 특정 → 원인 분석 → 수정안 도출 → 수정 검증 흐름을 템플릿화합니다.
- **논리 추론(Logical Reasoning)**: 전제 정리 → 관계 식별 → 연역 적용 → 결론 도출 형태로 구조화합니다.

## 성능 최적화(Performance Optimization)
- **추론 패턴 캐싱(Caching)**: 문제 임베딩 유사도를 기반으로 기존 추론을 재사용해 비용/시간을 줄이는 아이디어를 제시합니다.
- **적응형 추론 깊이(Adaptive Depth)**: 초기 단계 수로 시도 후, 해답이 불완전하면 **추론 단계 수를 점진적으로 증가**시키는 전략을 설명합니다.

## 평가 지표(Evaluation Metrics)
- 추론 사슬에 대해 **일관성(coherence), 완전성(completeness), 정답성(correctness), 효율성(efficiency), 명료성(clarity)** 등을 측정하는 예시 함수를 제시합니다.

## 모범 사례(Best Practices)
- 번호 매기기 등 **명확한 단계 표식**
- 중간 과정을 생략하지 않는 **과정 공개**
- 계산/논리의 **명시적 검증**
- **가정 명시**, **엣지 케이스** 점검
- 예시로 패턴을 먼저 보여주는 **샘플 기반 유도**

## 흔한 함정(Common Pitfalls)
- 성급한 결론, 순환 논리, 중간 단계 누락, 불필요하게 복잡한 단계 추가, 형식의 일관성 붕괴를 경고합니다.

## CoT 사용 가이드(When to Use)
- **권장**: 수학/산술, 논리 추론, 다단계 계획, 코드 생성·디버깅, 복잡 의사결정
- **비권장**: 단순 사실 질의, 단순 조회, 창작 중심, 간결성이 최우선, 지연에 민감한 실시간 시나리오
