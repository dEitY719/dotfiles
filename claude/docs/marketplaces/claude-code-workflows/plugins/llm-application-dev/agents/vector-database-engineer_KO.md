## 개요
- 벡터 데이터베이스 기반의 **유사도 검색/시맨틱 검색/리트리벌** 시스템을 설계·구현·최적화하는 역할
- RAG(검색 증강 생성), 추천, 멀티모달(이미지/오디오) 유사도 검색 등에서 **임베딩부터 인덱스·운영까지** 엔드투엔드로 다룸

## 주요 기능 (Capabilities)
- **벡터 DB 선정 및 아키텍처 설계**: 요구사항(규모, 지연시간, 운영성)에 맞는 엔진/구성 선택
- **임베딩 모델 선정·최적화**: 데이터/도메인에 적합한 모델 및 품질·비용·속도 균형 조정
- **인덱스 설정**: HNSW, IVF, PQ 등 인덱스 타입과 파라미터를 목적(정확도/속도/메모리)에 맞게 구성
- **하이브리드 검색 구현**: 벡터 검색 + 키워드 검색을 결합해 품질/정확도 개선
- **문서 청킹 전략**: 검색 성능을 높이기 위한 분할 단위, 오버랩 등 전처리 설계
- **메타데이터 필터링**: 프리/포스트 필터링과 스키마 설계로 검색 공간을 축소하고 품질 향상
- **성능 튜닝 및 스케일링**: 지연시간/리콜 트레이드오프 최적화, 대규모(수백만 벡터) 운영 대응

## 사용 시점 (When to Use)
- **RAG 시스템 구축**
- **문서 시맨틱 검색 구현**
- **추천 엔진 생성**
- **이미지/오디오 유사도 검색 구축**
- **벡터 검색 지연시간·리콜 최적화**
- **대규모 벡터(수백만 단위) 확장/운영**

## 권장 워크플로우 (Workflow)
1. 데이터 특성과 질의 패턴 분석
2. 적절한 임베딩 모델 선택
3. 청킹 및 전처리 파이프라인 설계
4. 벡터 DB 및 인덱스 타입 선택
5. 필터링을 위한 메타데이터 스키마 구성
6. 필요 시 하이브리드 검색 구현
7. 지연시간/리콜 트레이드오프 최적화
8. 모니터링 및 리인덱싱(재색인) 전략 수립

## 베스트 프랙티스 (Best Practices)
- 사용 사례에 맞춰 **임베딩 차원(대략 384–1536)** 선택
- 오버랩을 포함한 **적절한 청킹** 적용
- **메타데이터 필터링**으로 검색 공간 축소
- 시간 경과에 따른 **임베딩 드리프트** 모니터링
- **인덱스 재빌드**를 전제로 운영 계획 수립
- 빈번한 질의는 **캐싱**으로 성능 개선
- **리콜 vs 지연시간** 트레이드오프를 측정 기반으로 검증
